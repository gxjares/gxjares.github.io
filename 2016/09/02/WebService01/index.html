WebService

发表于 2016-09-02   |   分类于 JavaWeb

WebService也叫做Xml Web Service WebService，是一种可以接收从Internet或者Internet上的其它系统中传递过来的请求，轻量级的独立的通讯技术。是通过SOAP在Web上提供的软件服务，使用WSDL文件进行说明，并通过UDDI进行注册。

可以这么认为：WebService被定义用来使不同应用之间通过网络传输数据的一种标准，该标准与具体的语言无关。比如有两个应用（A和B），A应用使用php开发，B项目使用java开发：

  1. A应用提供接口（B发送请求数据给应用）
    B调用A接口（B发送请求数据） -> A接口接收 -> A接口处理B的数据

  2. B应用提供接口（A应用需要从B应用获取数据来做展现）
    A调用B接口（提供相应的查询参数） -> B接收 -> B返回数据（若B中的数据有更新，则A取到的数据也是最新的）

WebService相关的四种技术

1. XML

XML是用来标记数据的（对于XML标签详细介绍，可以参考W3C），如上所说，webservice不依赖于某一门特定的编程语言。而不同的应用系统可能是使用不同的编程语言来开发，（如上：项目A使用PHP开发，项目B使用Java开发）所以说就需要一种方式来交换数据，这种方式同样不依赖于某种编程语言，大多数软件都集成了XML标签，因此使用XML格式的数据来实现数据的交换。XML面向短期的临时数据处理、面向万维网络，是Soap的基础。

2. SOAP

Simple Object Access Protocol 简单对象存取协议，是WebService的通信协议。当用户通过UDDI找到你的描述文档后，可以通过SOAP调用你建立的Web服务中的一个或多个操作。SOAP是XML文档形式的调用方法的规范，可以支持不同的底层接口，像http(s)或者smtp。当调用方一旦发现提供Webservice接口的地址，它将会使用SOAP协议和提供Webservice的系统进行连接。

3. WSDL

Web Services Description Language，WSDL文件是一个XML文档，用于说明一组SOAP消息以及如何交换这些消息。大多数情况下由软件自动生成和使用。当有客户端发起调用的时候，Webservice提供方会参考这个WSDL规则来验证数据请求的合法性。

4. UDDI

Universal Description, Discovery, and Integration，是一个主要针对Web服务供应商和使用者的新项目。在用户能够调用Web服务之前，必须确定这个服务内包含哪些商务方法，找到被调用的接口定义，还要在服务端来编制软件，UDDI是一种根据描述文档来引导系统查找相应服务的机制。UDDI利用SOAP消息机制（标准的XML/HTTP）来发布、编辑、浏览以及查找注册信息。它采用XML格式来封装各种不同类型的数据，并且发送到注册中心或者由注册中心来返回需要的数据。UDDI列出了哪些服务是可以被访问的，当一个系统需要数据的时候，首先它会先查找UDDI，找出从除了自己之外的那些可以获取到自己需要的数据的服务，然后进行链接获取数据。这一点有点类似于DNS的功能。

调用原理

Web服务有两层含义：

  1. 指封装成单个实体并发布到网络上的功能集合体
  2. 指功能集合体被调用后所提供的服务，也就是说，Web服务是一个URL资源，客户端可以通过编程方式请求得到它的服务，而不需要知道所请求的服务是怎样实现的，这一点和传统的分布式组件对象模型不同。

Web服务的体系结构基于三个角色：

  • Web服务的提供者
  • Web服务的调用者
  • Web服务中介

这三个角色通过发布、发现、绑定三个动作构建web服务的体系结构。

实现一个完整的web服务的步骤

  • Web服务提供者设计实现Web服务，调试正确后通过Web中介者发布，并在UDDI注册中心注册。（发布）
  • Web服务请求者向Web服务中介者请求特定的服务，中介者根据请求查询UDDI注册中心，为请求者寻找特定的服务。（发现）
  • Web服务中介者找到特定的服务后向Web服务请求者返回满足条件的Web服务描述信息（WSDL写成，各种支持Web服务的机器都能阅读）。（发现）
  • 利用Web服务返回的描述信息生成相应的SOAP消息，发送给Web服务提供者，即可实现Web服务的调用。（绑定）
  • Web服务提供者按SOAP消息执行相应的操作，并将服务执行结果返回给Web服务请求者。（绑定）
