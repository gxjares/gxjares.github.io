java_web some issues

发表于 2016-07-12   |   分类于 JavaWeb

记录一些 java web 的知识点

1.提交表单后在action中转到下一个jsp时用<%=request.getParameter(“username”) %>可以取到值，但EL表达式取不到值

EL表达式是获取：application、session、request、page中的attribute的值的。get的是parameters（集合?），我这样写获得应该是request里的attribute，所以EL改成${param.username} —-(集合param中具体的attribute)就可以获取参数的值了。

2.请求参数包含中文

请求参数中有值为中文时，需要使用URLEncoder.encode(“中文参数值”, “UTF-8”);对其进行编码后再传值，但此时在jsp页面获取该参数值后会出现乱码，需要<%=URLDecoder.decode(new String(request.getParameter(“username”).getBytes(“ISO8859-1”), “UTF-8”)) %>解码：tomcat默认以ISO8859-1二进制存储中文，先获取二进制的字节码生成UTF-8格式的字符串，然后解码处理。

4.Struts1 & Struts2

与Struts1不同，Struts2对用户的每一次请求都会创建一个Action，所以Struts2中的Action是线程安全的。 Struts1是将Action放入缓存，每次请求会从缓存中获取Action来处理，在应用生命周期内仅会存在一个实例，相当于单例。
