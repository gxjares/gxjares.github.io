冒泡排序（邻居好说话）

冒泡排序的基本思想是：每次相邻两个元素，如果他们的顺序不符合要求，则调换他们的顺序。
要排序{ 1, 3, 5, 2, 4, 0, 8, 7, 12, 11, 10 }，说明越小的数越靠前。首先比较前两位，发现1比3小，则不调换顺序；继续比较3和5，不调换；继续比较5和2，调换；以此类推，第一趟下来的结果为：{ 1, 3, 2, 4, 0, 5, 7, 8, 11, 10, 12 }。继续排序：
{ 1, 3, 2, 4, 0, 5, 7, 8, 11, 10, 12 }
{ 1, 2, 3, 0, 4, 5, 7, 8, 10, 11, 12 }
{ 1, 2, 0, 3, 4, 5, 7, 8, 10, 11, 12 }
{ 1, 0, 2, 3, 4, 5, 7, 8, 10, 11, 12 }
{ 0, 1, 2, 3, 4, 5, 7, 8, 10, 11, 12 }

冒泡排序的核心是双重嵌套循环，可以看出其时间复杂度是O(N²)，这是一个非常高的时间复杂度。早在1956年就有人研究尝试改进，但是结果却不如人意。正如Donald E. Knuth所说：“冒泡排序除了它迷人的名字和导致了某些有趣的理论问题这一事实
之外，似乎没有什么值得推荐的。”

快速排序（最常用）

我们继续对{ 1, 3, 5, 2, 4, 0, 8, 7, 12, 11, 10 }进行排序，首先在这个序列中找一个基准数（即一个参照数），假设为5，然后所有比这个数大的放右边，比这个数小的放左边。初始的时候5在第三个位置，现在就要找到这个位置，并且以该位置为分界点，左边都小于5，右边都大于5。方法很简单：分别从序列的两端开始找，先从右往左找到第一个小于等于5的数字（即0），然后从左往右找到第一个比5大的数字，发现越过了0，即从右向左和从左向右相遇了，说明从左向右“越界”，查找结束，这时候交换0和5（基准数），得到{ 1, 3, 0, 2, 4, 5, 8, 7, 12, 11, 10 }，接着查找，从右向左找“小”数
（即5）,和基准数一致，结束。左边{1, 3, 0, 2, 4}，右边{8, 7, 12, 11, 10}，对于这两个子序列继续使用该方法（递归），比如左边{1, 3, 0, 2, 4}:设置基准数为2，从右向左找“小”数（2），从左向右找“大”数（3），交换2和3:{1, 2, 0, 3, 4}，从右向左找“小”数（0）,从左向右找“大”数（3），越界，交换0和基准数2:{1, 0, 2, 3, 4}，OK。

快速排序的每一轮处理其实就是将基准数归位，直到所有的数都归位为止。
快速排序之所以比较快，是因为相比冒泡排序，每次交换是跳跃式的。每次排序的时候设置一个基准点，将小于等于基准点的数全部放到基准点的左边，将大于等于基准点的数全部放到基准点的右边。这样在每次交换的时候就不会像冒泡排序一样只能在相邻的数之间进行交换，交换的距离就大得多了。因此总的比较和交换次数就少了，速度自然就提高了。当然在最坏的情况下，仍可能是相邻的两个数进行了交换。因此快速排序的最差时间复杂度和冒泡排序是一样的，都是O(N2)，它的平均时间复杂度为O (NlogN)。其实快速排序是基于一种叫做“二分”的思想。
